Heavy Lifting

React Vite Tailwind CSS IndexedDB Claude AI PWA
Launch App View on GitHub

Overview

Heavy Lifting is a mobile-first Progressive Web App that plans, logs, and intelligently adapts your workouts over time. It combines a pure-logic workout generator (21 tested cases) with optional Claude AI enhancement—no account required, all data stored locally on your device.

The Problem

Most gym tracking apps are either too simple (just a notepad) or too complex (subscription plans, synced accounts, overwhelming UX). I wanted something that actually tells you what to do next based on what you did last time—adapting to your performance without requiring any manual configuration or a monthly fee.

The Solution

A three-layer weight recommendation engine: your last logged weight as a baseline, an ease modifier based on how you rated each set (easy/just-right/hard), and an optional Claude AI overlay capped at ±15% of the calculated value. The AI never overrides the data—it refines it. The app works fully offline via PWA with IndexedDB storage and can be installed directly to your phone's home screen.

Key Features

  • Smart workout generation — PPL, upper/lower, and full-body rotation with auto deload detection
  • Set-by-set logging with rest timer, PR celebrations, and mid-session exercise swaps
  • Progress charts (lift curve + volume) and all-time PR history via Recharts
  • Gym & equipment manager — filter exercises to what's actually available at your gym
  • XP system, levels, and streaks for habit-building gamification
  • 130+ exercise library across push, pull, legs, and core categories
  • Installable PWA — works offline, no app store needed

Technical Stack

Frontend: React 18, Vite, Tailwind CSS v3, Recharts for data visualization

Storage: Dexie.js (IndexedDB wrapper) — fully client-side, no backend

AI: Anthropic Claude API (optional, user-supplied key) with deterministic fallback

Testing: Vitest — 21 unit tests covering workout generation, weight calc, and rotation logic

Deployment: Netlify + vite-plugin-pwa for service worker and offline caching

What I Learned

The most interesting design challenge was figuring out when to trust the AI vs. the data. The answer: always anchor to the data first and treat the AI as a bounded suggestion layer. This pattern—deterministic core with AI as an optional enhancement—made the app reliable enough to actually use daily while still feeling intelligent. I also learned that mobile performance is entirely different from desktop: the 100dvh bug, IndexedDB in private browsing, and input zoom on iOS all required specific fixes that only show up when you test on a real device.
