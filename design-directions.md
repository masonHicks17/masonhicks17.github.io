# Final Design Direction for Mason Hicks Portfolio

## Direction Summary

The final direction is a refined, modern portfolio that keeps the sharper elegant qualities from the strongest mockup while shifting the palette toward cream, parchment, charcoal, and muted olive.

This should feel:

- modern
- tech-oriented
- masculine
- premium but restrained
- appropriate for a student portfolio
- strong on both desktop and mobile

The site should keep a recognizable portfolio structure:

- hero
- about
- projects
- workflow
- experience
- contact

Dark mode remains part of the final system and should feel designed, not treated as an afterthought.

## Visual Identity

The visual identity should sit between product design portfolio and polished systems-minded personal site.

Key traits:

- warm, creamy light backgrounds instead of stark white
- structured layouts with clean spacing and strong hierarchy
- charcoal typography for credibility and clarity
- muted olive or sage as the primary accent
- subtle brass or tan used sparingly for warmth
- elegant, modern interfaces rather than glossy startup aesthetics
- calm motion and quiet detail instead of flashy visual tricks

This should not feel like:

- a luxury fashion editorial site
- a neon AI landing page
- a generic SaaS template
- a dark, overly aggressive fitness brand

## Light Mode Palette

Use this as the default visual flagship:

- background: `#F6F1E7`
- surface: `#EDE5D8`
- elevated surface: `#FBF8F2`
- primary text: `#1F2421`
- muted text: `#686960`
- border: `#D4CCBE`
- accent: `#6F7B5C`
- accent strong: `#7E8B68`
- optional secondary accent: `#A78E6A`

Usage guidance:

- Background should feel warm cream, not yellow.
- Surfaces should read as parchment or soft beige, not muddy tan.
- Olive should be present enough to define the site, but still restrained.
- Brass or tan should appear only in small accents, tags, dividers, or subtle emphasis.

## Dark Mode Palette

Dark mode should feel warm, grounded, and readable:

- background: `#171A18`
- surface: `#232824`
- elevated surface: `#2A302B`
- primary text: `#F1ECE2`
- muted text: `#B4B0A6`
- border: `#3A4139`
- accent: `#8E9B76`
- accent strong: `#9EAC84`
- optional secondary accent: `#B29A73`

Usage guidance:

- Avoid pure black and pure white.
- Keep contrast strong enough for accessibility, but softer than stark monochrome.
- Olive in dark mode can be slightly brighter than light mode so the interface still feels alive.

## Typography Direction

Typography should keep the portfolio feeling intelligent and composed.

Recommended direction:

- Display type: keep a refined serif for selective emphasis, large section headlines, or hero moments.
- Body/UI type: use a clean modern sans-serif for navigation, body copy, metadata, and buttons.
- Serif should be used with restraint so it feels sharp and intentional.
- Sans-serif should carry most of the site's structure and clarity.

Tone goals:

- sharper than soft editorial
- calmer than aggressive startup branding
- more mature than a typical student portfolio

Typography behavior:

- Strong hero headline hierarchy.
- Slightly tighter body copy than the current build.
- More disciplined contrast between section titles, card titles, and metadata.
- Buttons, tags, and small labels should feel precise and understated.

## Spacing and Layout

Spacing should create a premium sense of control without making the site feel sparse.

Layout principles:

- Keep the familiar single-page portfolio flow.
- Increase hierarchy between major sections instead of giving every section the same rhythm.
- Use more breathing room in hero and featured work.
- Keep content widths disciplined for readability.
- Let flagship project areas feel more intentional and spacious than supporting sections.

Desktop:

- Strong horizontal composition in the hero.
- Featured project cards can have more differentiated hierarchy.
- Section transitions can use subtle shifts in background tone or surface treatment.

Mobile:

- Preserve the same tone, but reduce visual density.
- Keep spacing generous without making users scroll through oversized empty gaps.
- Prioritize strong stacking and clean card grouping.

## Cards and Surfaces

Cards should move away from generic rounded-template styling and feel more like composed content panels.

Direction:

- Soft parchment or graphite surfaces depending on theme.
- Light borders with quiet contrast.
- Rounded corners are fine, but should be controlled and not overly playful.
- Shadows should stay subtle and premium, never floaty or glossy.
- Featured cards can use slightly different scale or composition to create hierarchy.

Project and game visuals:

- Replace placeholder gradients with reusable screenshot or mockup containers.
- Containers should work even before final imagery is available.
- Visual placeholders should feel like intentional media frames, not filler blocks.

## Buttons and UI Details

Buttons should feel modern and assured.

Primary buttons:

- Olive accent background in light mode.
- Slightly brighter sage/olive in dark mode.
- Charcoal or warm off-white text depending on contrast.

Secondary buttons:

- Neutral surface with border emphasis.
- More understated than the current orange-heavy treatment.

UI detail guidance:

- Tags should use muted parchment or olive-tinted surfaces.
- Dividers and borders should do more of the visual work than loud gradients.
- Accent color should be consistent across buttons, tags, active states, and key highlights.

## Imagery Style

Imagery should support Mason's identity as a practical builder, not distract from it.

Homepage hero imagery:

- Premium mockup-driven composition
- cream and neutral framing
- structured UI, charts, systems overlays
- calm, product-forward atmosphere

Project visuals:

- Strong mobile or interface mockups
- screenshot containers that feel reusable and intentional
- especially important for Heavy Lifting, which should feel like the flagship proof point

Avoid:

- loud abstract blobs
- cyberpunk AI motifs
- fitness cliches
- stock-photo laptop desk scenes
- overly glossy 3D effects

## Why This Fits Mason Hicks

This direction fits Mason Hicks' portfolio because it balances credibility, ambition, and taste.

It works well for Mason specifically because:

- it keeps the site clearly professional and portfolio-appropriate
- it supports a systems-minded, AI-assisted builder identity
- it gives Heavy Lifting a stronger stage without turning the whole site into a fitness brand
- it feels more distinct than a standard student portfolio without becoming overly experimental
- it can be implemented by Claude without a risky structural rewrite
- it supports both desktop polish and mobile clarity

Most importantly, it gives the portfolio a more authored point of view. The site will still be approachable and readable, but it should feel more deliberate, more premium, and more memorable.
